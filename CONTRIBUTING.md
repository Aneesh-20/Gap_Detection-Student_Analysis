# Contributing to Gap Detection

First off, thanks for taking the time to contribute! 🎉

## Development Setup
Please refer to the `README.md` for local setup instructions using Docker or manual installation.

## Branching Strategy
- `main` is the primary branch.
- Create feature branches named `feat/your-feature-name` or `fix/your-bug-fix`.

## Pull Requests
1. Ensure all tests pass (`pytest backend/tests`).
2. Update the documentation if you are adding a new feature.
3. Submit a PR describing the changes and linking any relevant issues.
